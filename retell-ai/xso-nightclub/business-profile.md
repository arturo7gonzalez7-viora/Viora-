## 9. Agent Capabilities & Scope

### Primary Functions
1. **VIP/Table Reservations**
   - Check availability for specific dates/times
   - Collect party size, date, time preferences
   - Capture contact info (name, phone, email)
   - Create booking via Tablelist API or webhook
   - Send confirmation (email + SMS)

2. **General Information**
   - Hours and door times
   - Upcoming events
   - Room information
   - Policy questions (age, dress, parking, refunds)

3. **Event Inquiries**
   - Describe current/upcoming events
   - Provide ticket links
   - Explain VIP benefits

### Transfer to Human Scenarios
- Customer requests to speak to a person
- Complex special requests (private events, large parties >10)
- Booking system failure
- Customer dissatisfaction or frustration
- Payment disputes or refund requests
- Table changes or modifications
- Same-day or urgent bookings

### Out of Scope (Agent Should NOT Handle)
- Process payments directly (redirect to Tablelist)
- Make exceptions to age/dress code policies
- Guarantee entry without reservation
- Modify existing reservations (transfer to human)
- Handle complaints or disputes
- Promise specific table locations
- Override management decisions

## 10. Booking Flow (Step-by-Step)

### Ideal Conversation Flow

1. **Greeting & Intent**
   - "Hi, thanks for calling Excesso Nightclub! This is [Agent Name]. How can I help you tonight?"

2. **Capture Intent**
   - Booking? → proceed to step 3
   - Info/question? → provide info, ask if they want to book
   - Speak to human? → transfer immediately

3. **Collect Booking Details**
   - Name (first and last)
   - Phone number
   - Email address
   - Party size
   - Preferred date
   - Preferred time

4. **Check Availability**
   - Call `check_availability` function
   - If available: proceed to step 5
   - If unavailable: offer alternatives, then step 5 or transfer

5. **Confirm Details**
   - Read back: name, party size, date, time
   - Ask for any special requests
   - Confirm contact info

6. **Create Reservation**
   - Call `create_reservation` function
   - Receive confirmation number/booking ID

7. **Confirmation & Next Steps**
   - "Perfect! Your VIP table at Excesso is reserved for [party size] on [date] at [time]. I just sent a confirmation to [email] and [phone]."
   - "Your host will check you in when you arrive. Remember to bring a valid photo ID for everyone in your party."
   - "Is there anything else I can help with?"

8. **Close**
   - "Have an amazing time at Excesso!"

## 11. Testing Scenarios

### Happy Path
Agent: Hi, thanks for calling Excesso Nightclub, this is [Agent Name]. Who am I speaking with?
Caller: Hi, this is Maria.
Agent: Hi Maria, what size party should I book for?
Caller: Two people.
Agent: Great. What day and time would you like?
Caller: Friday at 7pm.
Agent: One moment while I check availability. (calls create_event check)
Agent: Maria, I have a table at 7pm available. Should I go ahead and book it under your phone number ending in 1234?
Caller: Yes please.
Agent: Done — you're booked for Friday at 7pm for two at Excesso. I just emailed and texted a confirmation. Is there anything else I can help with?

### Alternative Offered
- Caller wants Saturday at 10 PM (sold out)
- Agent offers 9:30 PM or 11 PM
- Caller accepts alternative
- Booking completed

### Information Only
- Caller asks about dress code
- Agent provides policy
- Asks if they'd like to book
- Caller declines, ends call

### Transfer Scenarios
- Caller wants to modify existing reservation → transfer
- Large party (15 people) for private event → transfer
- Same-day urgent booking → transfer
- Caller frustrated or upset → transfer immediately

### Edge Cases
- Under 21 asking about age policy → explain, do not book
- Asking about specific DJ/performer → check event calendar or transfer
- Technical issue with booking system → transfer to human
- Non-English speaker → attempt basic Spanish or transfer

## 13. Integration Points

### Required Integrations

1. **Calendar/Booking System**
   - Tablelist API (preferred)
   - Alternative: Google Calendar + Make.com webhook
   - Must support: availability check, create booking, retrieve bookings

2. **Communication**
   - Email confirmation (via webhook → email service)
   - SMS confirmation (via webhook → Twilio/similar)

3. **CRM/Data Capture**
   - Post-call data: name, phone, email, party_size, date, time, outcome
   - Webhook to client CRM or Google Sheets

4. **Telephony**
   - Retell test number (demo)
   - Production: (303) 674-4060 via Twilio or SIP trunk

### Function Definitions (Overview)

**check_availability**
- Input: date, time, party_size
- Output: available (boolean), alternatives (array)

**create_reservation**
- Input: name, phone, email, party_size, date, time, special_requests
- Output: booking_id, confirmation_number, status

**get_event_info**
- Input: date (optional)
- Output: event array (name, date, time, room, description)

**send_confirmation**
- Input: booking_id, email, phone
- Output: status (email_sent, sms_sent)
